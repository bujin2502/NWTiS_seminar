Biblioteka podržava kako starije binarne formate temeljene na OLE2 standardu (XLS, DOC, PPT), tako i moderne OOXML formate uvedene s Microsoft Officeom 2007 (XLSX, DOCX, PPTX). Drugi primjer prikazuje generiranje Excel izvještaja s primjenom stilova, ugrađenih formula i automatskog prilagođavanja širine stupaca. \chapter{Uvod}

U suvremenom okruženju razmjena podataka putem Microsoft Office dokumenata je nešto što se podrazumijeva. Excel tablice, Word dokumenti i PowerPoint prezentacije postali su de-facto standard za izvještavanje, analizu podataka i komunikaciju. Ručna obrada takvih dokumenata u sustavima koji zahtijevaju automatizaciju, poput sustava za upravljanje sadržajem, poslovnih aplikacija ili alata za ekstrakciju podataka, nije praktična ni skalabilna.

Apache POI projekt, koji razvija i održava Apache Software Foundation, nudi rješenje za ovaj problem u obliku besplatne Java biblioteke otvorenog koda. Biblioteka programerima omogućuje čitanje, pisanje i izmjenu Microsoft Office datoteka isključivo korištenjem Java koda, bez potrebe za instaliranim Microsoft Officeom. Time se postiže potpuna platformska neovisnost, što je posebno vrijedno u poslovnim okruženjima koja se oslanjaju na Java ekosustav.
 
Motivacija za odabir Apache POI-a kao teme ovog rada leži upravo u njegovoj rasprostranjenosti i praktičnoj primjenjivosti. Biblioteka je jedna od najkorištenijih Java biblioteka za rad s Office formatima i aktivno se razvija već više od dva desetljeća. Podržava kako starije binarne formate temeljene na OLE2 standardu (XLS, DOC, PPT), tako i moderne XML-bazirane OOXML formate uvedene s Microsoft Officeom 2007 (XLSX, DOCX, PPTX).

Cilj ovog rada je sustavno prikazati arhitekturu i ključne komponente Apache POI projekta te demonstrirati njegovu primjenu na konkretnim primjerima. Rad počinje pregledom povijesti i arhitekture projekta, zatim obrađuje rad s pojedinim formatima, a završava praktičnim primjerima generiranja dokumenata.

\chapter{Pregled Apache POI projekta}

\section{Povijest i razvoj}

Začeci Apache POI projekta sežu u travanj 2001. godine, kada je programer Andrew Oliver dobio kratkoročni ugovor za generiranje Excel izvještaja u Javi. Suočen s drastičnim porastom cijene komercijalne biblioteke koju je dotad koristio, Oliver je odlučio razviti vlastito rješenje otvorenog koda. Uz pomoć Marca Johnsona, kojeg je pronašao putem lokalne Java korisničke grupe, implementirana je podrška za OLE 2 \textit{Compound Document Format} kao temelj cijelog projekta. Johnson je preuzeo razvoj niskorazinskog sloja za rukovanje OLE2 strukturom, dok je Oliver paralelno radio na parsiranju XLS formata. Već s prvim izdanjem, POI 1.0, projekt je nadišao izvorne planove, uz podršku za čitanje i pisanje Excel datoteka, dodan je i serijalizacijski okvir za integraciju s drugim sustavima \cite{apacheApachePOIHistory}.

Kako je projekt rastao, postalo je jasno da nadilazi okvire jednog namjenskog alata. Nakon neuspjelog pokušaja integracije isključivo u Apache Cocoon projekt, donesena je odluka da se jezgreni POI API-ji prenesu u okvir Apache Jakarta projekta, dok su pojedine komponente donirane drugim projektima iz Apache ekosustava. U sklopu Jakarte, projekt je nastavio rasti zahvaljujući doprinosima zajednice, proširujući podršku na nove formate i slučajeve korištenja. Početkom 2007. godine POI je zaprovodio proces inkubacije i postao samostalni projekt najviše razine unutar \textit{Apache Software Foundation} \cite{apacheApachePOIHistory}.

Naziv projekta izvorno je bio akronim za ``\textit{Poor Obfuscation Implementation}'', humoristična referenca na činjenicu kako su Microsoftovi binarni formati, unatoč naizglednoj namjernoj kompliciranosti, bili uspješno reverzno razvijeni. Ista logika imenovanja proteže se i na potprojekte poput HSSF (\textit{Horrible SpreadSheet Format}), HWPF (\textit{Horrible Word Processor Format}) i ostale. Kako je projekt počeo privlačiti poslovne korisnike, objašnjenja akronima tiho su uklonjena s službenih stranica \cite{wikipediaApacheWikipedia}.

\section{Pregled dostupnih klasa}

Apache POI biblioteka pruža iznimno bogat skup klasa koje pokrivaju gotovo sve aspekte rada s Office dokumentima. Za Excel datoteke, klase poput \texttt{XSSFWorkbook}, \texttt{XSSFSheet}, \texttt{XSSFCell} i \texttt{XSSFCellStyle} omogućuju kreiranje i formatiranje tablica, dok \texttt{XSSFChart} i \texttt{XDDFChartData} podržavaju generiranje grafova. Za streaming obradu velikih datoteka dostupne su \texttt{SXSSFWorkbook} i \texttt{SXSSFSheet}. Word dokumenti pokriveni su klasama \texttt{XWPFRun}, \texttt{XWPFDocument}, \texttt{XWPFParagraph}, \texttt{XWPFTable} i \texttt{XWPFHeaderFooter}, a za PowerPoint prezentacije koriste se \texttt{XMLSlideShow}, \texttt{XSLFSlide}, \texttt{XSLFTextShape} i \texttt{XSLFTable} \cite{apachePOIJavadoc}.

Uz rad s dokumentima, biblioteka nudi i infrastrukturne komponente. \texttt{POIFSFileSystem} i \texttt{OPCPackage} upravljaju niskorazinskim pristupom OLE2 i OOXML datotečnim strukturama. Klase iz paketa \texttt{EncryptionInfo}, \texttt{Encryptor} i \texttt{Decryptor} omogućuju enkripciju i dekripciju dokumenata. Za ekstrakciju teksta dostupne su \texttt{XWPFWordExtractor}, \texttt{ExcelExtractor} i \texttt{SlideShowExtractor}, a \texttt{DataFormatter} i \texttt{DateUtil} pomažu pri interpretaciji formata ćelija. Klase poput \texttt{EmbeddedExtractor} i \texttt{PictureData} omogućuju ekstrakciju ugrađenih slika i objekata iz dokumenata \cite{apachePOIJavadoc}.

\chapter{Rad s formatima datoteka}

Ovo poglavlje rada posvećeno je praktičnoj primjeni Apache POI biblioteke za rad s tri najzastupljenija Microsoft Office formata, Excel tablicama, Word dokumentima i PowerPoint prezentacijama. Za svaki format postoje dvije implementacije koje odgovaraju dvjema generacijama Office standarda: starija binarna implementacija temeljena na OLE2 formatu te novija implementacija temeljena na OOXML standardu. U nastavku su opisane ključne mogućnosti svake implementacije, uz osvrt na specifičnosti pojedinih slučajeva korištenja \cite{apacheApachePOIComponents}.

\section{Excel datoteke}

Za rad s Excel datotekama Apache POI nudi tri implementacije. HSSF (\textit{Horrible SpreadSheet Format}) pokriva stariji binarni .xls format koji odgovara Excel verzijama 97–2003, a XSSF (\textit{XML SpreadSheet Format}) namijenjen je modernom .xlsx formatu temeljenom na OOXML standardu. Oba modula podržavaju čitanje i pisanje, a dostupna je i zajednička SS apstrakcija koja omogućuje pisanje koda neovisnog o konkretnom formatu. Isti kod može generirati i .xls i .xlsx datoteku promjenom jednog parametra. Kroz ove module moguće je kreirati radne knjige i listove, postavljati vrijednosti ćelija, primjenjivati stilove i formate, koristiti formule te upravljati uvjetnim oblikovanjem \cite{apachePOIHSSFPOIXSSFSXSSF}.

Za slučajeve kada je potrebno generirati izrazito velike Excel datoteke, standardni XSSF pristup može biti problematičan zbog visokog memorijskog opterećenja, cijela struktura dokumenta drži se u memoriji. Kao rješenje, od verzije 3.8-beta3 dostupan je SXSSF (\textit{Streaming SpreadSheet Format}), streaming implementacija izgrađena povrh XSSF-a. SXSSF postiže mali memorijski otisak ograničavanjem pristupa samo redovima unutar kliznog prozora fiksne veličine, dok se stariji redovi automatski zapisuju na disk. Veličina prozora može se konfigurirati pri inicijalizaciji radne knjige ili zasebno po listu. Važno je napomenuti kako je SXSSF namijenjen isključivo pisanju, za čitanje velikih datoteka preporučuje se korištenje SAX-temeljenog Event modela koji XML strukturu .xlsx datoteke procesira inkrementalno \cite{apachePOIHSSFPOIXSSFSXSSF}.

\section{Word dokumenti}

Za rad s Word dokumentima Apache POI nudi dvije implementacije. HWPF (\textit{Horrible Word Processor Format}) pokriva stariji binarni .doc format koji odgovara Word verzijama 97–2003, a XWPF (\textit{XML Word Processor Format}) namijenjen je modernom .docx formatu temeljenom na OOXML standardu. HWPF podržava čitanje i ograničeno pisanje, a pruža i jednostavnu ekstrakciju teksta za još starije Word 6 i Word 95 formate. XWPF nudi stabilniji i potpuniji API te podržava čitanje i pisanje složenijih dokumenata \cite{apacheApachePOIHWPFXWPF}.

Glavna ulazna točka za rad s .docx datotekama je klasa XWPFDocument, iz koje se dohvaćaju svi elementi dokumenta. Paragrafi su dostupni kroz klasu XWPFParagraph, a svaki paragraf sastoji se od jednog ili više XWPFRun objekata koji predstavljaju tekstualne odsječke s jednakim stilskim svojstvima: fontom, veličinom, podebljavanjem, nakošenje slova i slično. Tablice se dohvaćaju kroz klasu XWPFTable, pri čemu su pojedini redovi i ćelije dostupni kroz XWPFTableRow i XWPFTableCell. Za ekstrakciju cjelokupnog teksta dokumenta u jednom koraku dostupna je klasa XWPFWordExtractor. Zaglavlja i podnožja dostupna su kroz XWPFHeader i XWPFFooter klase putem XWPFHeaderFooterPolicy objekta \cite{apacheApachePOIHWPFXWPF}.

\section{Ostale mogućnosti}

Uz osnovnu podršku za Excel i Word, Apache POI 
biblioteka nudi niz dodatnih mogućnosti koje pokrivaju specifične poslovne 
i tehničke potrebe.

\subsection{PowerPoint prezentacije}

Kroz XSLF API moguće je iterirati po svim oblicima na svakom slajdu i 
ekstrahirati tekstualni sadržaj, što je korisno za indeksiranje 
prezentacija. Metoda \texttt{draw()} klase \texttt{XSLFSlide} omogućuje 
iscrtavanje cijelog slajda na \texttt{java.awt.Graphics2D} objekt, čime 
je moguće generirati PNG ili JPEG slike pojedinih slajdova bez 
instaliranog PowerPointa. Ova mogućnost koristi se u sustavima za pregled 
prezentacija u web preglednicima, gdje je potrebno prikazati slajdove 
bez instaliranog Microsoft Officea na poslužitelju. Uz to, XSLF API 
podržava dodavanje slika, tablica i tekstualnih okvira na slajdove, 
postavljanje tranzicija te upravljanje slide masterom i layoutima 
\cite{apachePOIHSLFQuick}.

\subsection{Enkripcija i dekripcija dokumenata}

Apache POI podržava enkripciju i dekripciju Office dokumenata standardnim 
OOXML mehanizmom zaštite lozinkom. Enkripcija se provodi kroz klase 
\texttt{EncryptionInfo} i \texttt{Encryptor}, pri čemu se dokument zapisuje 
u \texttt{POIFSFileSystem} kontejner koji je šifriran odabranim algoritmom. 
Podržani algoritmi uključuju AES-128 i AES-256 koji se koriste u modernim 
verzijama Microsoft Officea. Dekripcija se provodi analognim postupkom kroz 
klasu \texttt{Decryptor} — dokument se učitava kao \texttt{POIFSFileSystem}, 
provjerava se ispravnost lozinke i dohvaća dešifrirani tok podataka iz kojeg 
se rekonstruira radna knjiga ili dokument \cite{apachePOIJavadoc}.

\subsection{Ekstrakcija teksta}

Za slučajeve kada nije potrebna puna manipulacija dokumentom nego samo 
dohvat tekstualnog sadržaja, Apache POI nudi specijalizirane klase za 
ekstrakciju teksta. \texttt{XWPFWordExtractor} ekstrahira sav tekst iz 
\texttt{.docx} datoteke uključujući paragrafe, tablice i zaglavlja, 
\texttt{XSSFExcelExtractor} radi analogno za \texttt{.xlsx} datoteke, 
a \texttt{SlideShowExtractor} za \texttt{.pptx} prezentacije. Sve tri 
klase implementiraju zajednički \texttt{POITextExtractor} sučelje, što 
omogućuje uniforman pristup ekstrakciji bez obzira na format dokumenta. 
Ekstrakcija teksta posebno je korisna u sustavima za pretraživanje i 
indeksiranje sadržaja, kao što su Apache Lucene i Apache Solr, s kojima 
Apache POI ima izravnu integraciju putem Apache Tika biblioteke 
\cite{apachePOIJavadoc}.

\subsection{Rad s metapodacima}

Svaki Office dokument sadrži metapodatke poput naslova, autora, datuma 
kreiranja i izmjene te ključnih riječi. Apache POI omogućuje čitanje i 
pisanje tih metapodataka kroz \texttt{POIXMLProperties} klasu za OOXML 
formate, koja sadrži tri podskupa svojstava: osnovna svojstva 
(\texttt{CoreProperties}), proširena svojstva (\texttt{ExtendedProperties}) 
i prilagođena svojstva (\texttt{CustomProperties}). Za starije OLE2 
formate metapodaci su dostupni kroz \texttt{HPSF} komponentu 
(\textit{Horrible Property Set Format}). Ova mogućnost korisna je u 
sustavima za upravljanje dokumentima gdje je potrebno automatski 
popunjavati ili čitati metapodatke bez otvaranja samog dokumenta 
\cite{apachePOIJavadoc}.

\subsection{Podrška za Outlook i Visio}

Apache POI uključuje i module za formate koji su rjeđe zastupljeni u 
tipičnim poslovnim okruženjima. \texttt{HSMF} (\textit{Horrible Stupid 
Mail Format}) modul omogućuje čitanje Outlook \texttt{.msg} datoteka, 
pri čemu su dostupni pošiljatelj, primatelji, predmet, tijelo poruke i 
privici. Microsoft je 2007. godine otvorio specifikaciju ovog formata, 
što je dodatno olakšalo razvoj pouzdane implementacije 
\cite{apacheApachePOIComponents}.

Za Visio dijagrame dostupna su dva modula: \texttt{HDGF} pokriva stariji 
binarni \texttt{.vsd} format i podržava samo niskorazinsko čitanje i 
ekstrakciju teksta, dok \texttt{XDGF} pokriva moderniji XML-bazirani 
\texttt{.vsdx} format s nešto širom podrškom. Oba modula trenutno ne 
nude API za kreiranje novih Visio dijagrama programskim putem, već su 
usmjereni primarno na čitanje i ekstrakciju sadržaja 
\cite{apacheApachePOIComponents}.

\chapter{Praktični primjeri}

Za demonstraciju praktične primjene Apache POI biblioteke koristi se Eclipse IDE uz Maven kao alat za upravljanje ovisnostima. Maven omogućuje jednostavno upravljanje vanjskim bibliotekama kroz centralnu konfiguracijsku datoteku pom.xml, čime se izbjegava ručno preuzimanje i dodavanje JAR datoteka u classpath projekta \cite{apacheApachePOIdownload}.

Nakon kreiranja novog Maven projekta u Eclipseu putem File → New → Maven Project, u datoteku pom.xml potrebno je dodati dvije ovisnosti. Ovisnost poi pokriva podršku za starije binarne formate te predstavlja jezgrenu biblioteku projekta, dok ovisnost poi-ooxml dodaje podršku za moderne OOXML formate. Maven automatski preuzima sve tranzitivne ovisnosti, pa nije potrebno ručno upravljati dodatnim JAR datotekama. Trenutno aktualna stabilna verzija Apache POI biblioteke je 5.5.1. \cite{apacheApachePOIdownload}

\begin{lstlisting}[style=xml, caption={Ovisnosti u datoteci pom.xml}]
<dependency>
    <groupId>org.apache.poi</groupId>
    <artifactId>poi</artifactId>
    <version>5.5.1</version>
</dependency>
<dependency>
    <groupId>org.apache.poi</groupId>
    <artifactId>poi-ooxml</artifactId>
    <version>5.5.1</version>
</dependency>
\end{lstlisting}

\section{Kreiranje Word dokumenta}

Kreiranje Word dokumenta u Apache POI-u temelji se na klasi \texttt{XWPFDocument}, koja predstavlja glavni objekt dokumenta. Iz nje se kreiraju svi ostali elementi, zaglavlja, podnožja, paragrafi i tablice \cite{apacheXWPFQuickGuide}.

\subsection{Zaglavlje i podnožje}

Zaglavlje i podnožje kreiraju se metodama \texttt{createHeader()} i \texttt{createFooter()} uz parametar \texttt{HeaderFooterType.DEFAULT}, koji označava da se primjenjuju na sve stranice dokumenta. Unutar zaglavlja i podnožja kreira se paragraf, a tekst se dodaje kroz \texttt{XWPFRun} objekt. Isti princip, paragraf koji sadrži jedan ili više \texttt{XWPFRun} objekata, koristi se za sve tekstualne elemente u dokumentu \cite{apacheXWPFQuickGuide}.

\begin{lstlisting}[style=java, caption={Obrada Word dokumenta}]
XWPFHeader zaglavlje = dokument.createHeader(HeaderFooterType.DEFAULT);
XWPFParagraph zaglavljeParagraf = zaglavlje.createParagraph();
zaglavljeParagraf.setAlignment(ParagraphAlignment.CENTER);
XWPFRun zaglavljeRun = zaglavljeParagraf.createRun();
zaglavljeRun.setText("Fakultet organizacije i informatike");
zaglavljeRun.setFontSize(10);
zaglavljeRun.setColor("888888");
\end{lstlisting}

\subsection{Paragrafi i stilovi}

{\sloppy
Paragrafi se kreiraju metodom \texttt{createParagraph()} na objektu dokumenta. Svaki paragraf može imati postavljeno poravnanje (\texttt{setAlignment()}), razmak prije ili nakon (\texttt{setSpacingBefore()}, \texttt{setSpacingAfter()}) te jedan ili više \texttt{XWPFRun} objekata koji nose stvarni tekst. Na razini \texttt{XWPFRun} objekta definiraju se stilska svojstva teksta: font (\texttt{setFontFamily()}), veličina (\texttt{setFontSize()}), boja (\texttt{setColor()}), podebljanje (\texttt{setBold()}) i kurziv (\texttt{setItalic()}). U primjeru su na ovaj način kreirani naslov, podnaslov, uvodni paragraf i završna napomena \cite{apacheXWPFQuickGuide}.
\par}

\begin{lstlisting}[style=Java, caption={Kreiranje naslova}]
XWPFParagraph naslov = dokument.createParagraph();
naslov.setAlignment(ParagraphAlignment.CENTER);
naslov.setSpacingAfter(200);
XWPFRun naslovRun = naslov.createRun();
naslovRun.setText("Popis studenata i ocjena");
naslovRun.setBold(true);
naslovRun.setFontSize(20);
naslovRun.setFontFamily("Calibri");
\end{lstlisting}

\subsection{Tablica}

Tablica se kreira metodom \texttt{createTable()} uz navođenje broja redova i stupaca. U primjeru je kreirana tablica s 6 redova i 3 stupca, prvi red služi kao zaglavlje tablice, a preostalih 5 redova sadrži podatke o studentima \cite{apacheXWPFQuickGuide}.

Zaglavlje tablice stilizira se plavom pozadinskom bojom (\texttt{setColor()}) i bijelim podebljanim tekstom. Svaka ćelija zaglavlja dohvaća se metodom \texttt{getCell()} iz odgovarajućeg reda, a tekst se dodaje na isti način kao i u paragrafima dokumenta, kroz \texttt{XWPFParagraph} i \texttt{XWPFRun} objekte \cite{apacheXWPFQuickGuide}.

\begin{lstlisting}[style=Java, caption=Kreiranje zaglavlja tablice]
XWPFTable tablica = dokument.createTable(6, 3);
tablica.setWidth("100%");

String[] zaglavljaStupaca = {"Ime i prezime", "JMBAG", "Ocjena"};
XWPFTableRow zaglavljeRed = tablica.getRow(0);
for (int i = 0; i < zaglavljaStupaca.length; i++) {
    XWPFTableCell celija = zaglavljeRed.getCell(i);
    celija.setColor("2E75B6");
    XWPFParagraph p = celija.getParagraphs().get(0);
    p.setAlignment(ParagraphAlignment.CENTER);
    XWPFRun r = p.createRun();
    r.setText(zaglavljaStupaca[i]);
    r.setBold(true);
    r.setColor("FFFFFF");
    r.setFontSize(11);
}
\end{lstlisting}

Redovi s podacima pune se iteracijom kroz dvodimenzionalno polje. Radi bolje čitljivosti, primijenjeno je izmjenično bojanje redova, parni redovi ostaju bijeli, a neparni dobivaju svijetloplavu pozadinsku boju \cite{apacheXWPFQuickGuide}.

\begin{lstlisting}[style=Java, caption=Punjenje tablice podacima]
for (int i = 0; i < podaci.length; i++) {
    XWPFTableRow red = tablica.getRow(i + 1);
    String boja = (i % 2 == 0) ? "FFFFFF" : "DEEAF1";
    for (int j = 0; j < podaci[i].length; j++) {
        XWPFTableCell celija = red.getCell(j);
        celija.setColor(boja);
        XWPFParagraph p = celija.getParagraphs().get(0);
        p.setAlignment(ParagraphAlignment.CENTER);
        XWPFRun r = p.createRun();
        r.setText(podaci[i][j]);
        r.setFontSize(11);
    }
}
\end{lstlisting}

}
dokument.close();

\subsection{Izrada cirkularnog dokumenta}

Na ovom mjestu opisuje se ideja izgradnje cirkularnog pisma uz pomoć \texttt{XWPFDocument} klase. Cirkularno pismo je tehnika masovne personalizacije dokumenta. Prije nego što se uopće krene s implementacijom, potrebno je kreirati predložak dokumenta koji sadrži rezervirana mjesta (placeholdere) za dinamičke podatke. U konkretnom primjeru, u Word dokumentu se koriste oznake \texttt{\{\{IME\}\}}, \texttt{\{\{PREZIME\}\}}, \texttt{\{\{JMBAG\}\}}, \texttt{\{\{KOLEGIJ\}\}},\texttt{\{\{DATUM\}\}}  i \texttt{\{\{OCJENA\}\}} koje će se kasnije zamijeniti stvarnim vrijednostima iz CSV datoteke kao izvora podataka. Ove oznake su lako prepoznatljive i ne sadrže razmake, što olakšava njihovu identifikaciju u kodu.

Predložak bi izgledao na sljedeći način:

\begin{mdframed}
Poštovani/a \texttt{\{\{IME\}\}} \texttt{\{\{PREZIME\}\}},

na kolegiju \texttt{\{\{KOLEGIJ\}\}} ste ostvarili ocjenu \texttt{\{\{OCJENA\}\}}.

JMBAG: \texttt{\{\{JMBAG\}\}}

Datum: \texttt{\{\{DATUM\}\}}

S poštovanjem,\\
Fakultet organizacije i informatike
\end{mdframed}

Za potrebu popune navedenog predloška, koristi se CSV datoteka koja sadrži podatke o studentima i njihovim ocjenama. Ova datoteka služi kao izvor podataka za zamjenu rezerviranih mjesta u predlošku, omogućujući generiranje personaliziranih dokumenata za svakog studenta.

\begin{lstlisting}[style=csv, caption={Datoteka studenti.csv}]
ime,prezime,jmbag,kolegij,ocjena
Pero,Kos,0123456789,Napredne WEB tehnologije i servisi,Izvrstan (5)
Franjo,Kovač,0123456790,Napredne WEB tehnologije i servisi,Vrlo dobar (4)
Maja,Perić,0123456791,Napredne WEB tehnologije i servisi,Izvrstan (5)
Luka,Novak,0123456792,Napredne WEB tehnologije i servisi,Dobar (3)
Sara,Babić,0123456793,Napredne WEB tehnologije i servisi,Vrlo dobar (4)
\end{lstlisting}

U Prilogu se nalazi Java kod koji demonstrira kako se koristi Apache POI za čitanje predloška, zamjenu rezerviranih mjesta s podacima iz CSV datoteke i generiranje personaliziranih Word dokumenata za svakog studenta. Ovaj primjer ilustrira praktičnu primjenu Apache POI biblioteke u kontekstu masovne personalizacije dokumenata, što je česta potreba u obrazovnim institucijama i poslovnim okruženjima.

Program se sastoji od dvije metode: \texttt{main()} koja upravlja ukupnim tokom izvođenja i pomoćne metode \texttt{zamijeniUParagrafu()} koja provodi zamjenu placeholdera unutar pojedinog paragrafa.

U metodi \texttt{main()} najprije se definiraju putanje do predloška i CSV datoteke te se automatski generira trenutni datum korištenjem \texttt{LocalDate.now()}. CSV datoteka čita se redak po redak korištenjem \texttt{BufferedReader} klase, pri čemu se prvi redak (zaglavlje) preskače zastavicom \texttt{prviRed}. Svaki sljedeći redak dijeli se po zarezu metodom \texttt{split('','')}, a vrijednosti se pohranjuju u zasebne varijable. Za svakog studenta predložak se učitava kao novi \texttt{XWPFDocument} objekt kako bi svaki generirani dokument krenuo od čistog predloška. Po završetku zamjene, dokument se sprema pod nazivom oblika \texttt{obavijest\_Ime\_Prezime.docx}.

Metoda \texttt{zamijeniUParagrafu()} rješava specifičan problem XWPF implementacije, Word dokument interno može razbiti jedan vidljivi tekst u više \texttt{XWPFRun} objekata, zbog čega jednostavna zamjena po pojedinom odsječku ne bi mogla prepoznati cijeli placeholder. Metoda najprije spaja tekstove svih odsječaka paragrafa u jedan string korištenjem \texttt{StringBuilder} klase, zatim provodi zamjenu svih placeholdera, a konačni tekst upisuje u prvi odsječak dok ostale odsječke prazni. Na taj način stilska svojstva prvog odsječka ostaju sačuvana, a sadržaj je ispravno popunjen.

\section{Generiranje Excel izvještaja}

Generiranje Excel datoteke u Apache POI-u temelji se na hijerarhiji triju objekata: \texttt{XSSFWorkbook} predstavlja cijelu radnu knjigu (engl. \textit{workbook}), \texttt{XSSFSheet} predstavlja jedan list unutar nje, a \texttt{Row} i \texttt{Cell} odgovaraju retku i ćeliji na listu. Svi se kreiraju redom, od radne knjige prema ćeliji \cite{apachePOISpreadsheetQuickGuide}.

\subsection{Stilovi i formatiranje}

Za razliku od rada s Word dokumentima gdje se stil veže uz \texttt{XWPFRun} objekt, u Excel modulima stil se definira kao zasebni \texttt{XSSFCellStyle} objekt koji se kreira na razini radne knjige i potom dodjeljuje pojedinim ćelijama. Jedan stil može se primijeniti na više ćelija, što je važno jer svaka radna knjiga podržava ograničen broj jedinstvenih stilova. Font se kreira odvojeno metodom \texttt{createFont()} na objektu radne knjige te potom veže uz stil metodom \texttt{setFont()} \cite{apachePOISpreadsheetQuickGuide}.

Boja pozadine ćelije postavlja se kombinacijom:

\begin{itemize}
    \item \texttt{setFillForegroundColor()}
    \item \texttt{setFillPattern()}
\end{itemize}

pri čemu se za jednobojno ispunjenje koristi \texttt{FillPatternType.SOLID\_FOREGROUND}. Boja se predaje kao \texttt{XSSFColor} objekt koji prima standardni \texttt{java.awt.Color}:

\begin{lstlisting}[style=java, caption={Kreiranje stilova ćelija}]
XSSFCellStyle stilZaglavlja = radnaKnjiga.createCellStyle();
stilZaglavlja.setFillForegroundColor(new XSSFColor(
        new java.awt.Color(46, 117, 182), new DefaultIndexedColorMap()));
stilZaglavlja.setFillPattern(FillPatternType.SOLID_FOREGROUND);
stilZaglavlja.setAlignment(HorizontalAlignment.CENTER);

XSSFFont fontZaglavlja = radnaKnjiga.createFont();
fontZaglavlja.setBold(true);
fontZaglavlja.setColor(new XSSFColor(
        new java.awt.Color(255, 255, 255), new DefaultIndexedColorMap()));
fontZaglavlja.setFontHeightInPoints((short) 11);
stilZaglavlja.setFont(fontZaglavlja);
\end{lstlisting}

\subsection{Popunjavanje tablice}

Redovi se kreiraju metodom \texttt{createRow()} na objektu lista, a metodom \texttt{createCell()} ćelije na retku. Vrijednost ćelije postavlja se metodom \texttt{setCellValue()}, koja je preopterećena i prima \texttt{String}, \texttt{double}, \texttt{boolean} ili \texttt{Date}. Numeričke vrijednosti potrebno je pohraniti kao broj (a ne kao string) kako bi formule mogle nad njima raditi ispravno \cite{apachePOISpreadsheetQuickGuide}.

U primjeru je primijenjeno isto izmjenično bojanje redova kao u Word tablici iz prethodne sekcije: parni redovi ostaju bijeli, a neparni dobivaju svijetloplavu pozadinsku boju.

\begin{lstlisting}[style=java, caption={Popunjavanje tablice podacima}]
for (int i = 0; i < podaci.length; i++) {
    Row red = list.createRow(i + 1);
    XSSFCellStyle stil = (i % 2 == 0) ? stilParni : stilNeparni;
    for (int j = 0; j < podaci[i].length; j++) {
        Cell celija = red.createCell(j);
        if (j == 3) {
            celija.setCellValue(Integer.parseInt(podaci[i][j]));
        } else {
            celija.setCellValue(podaci[i][j]);
        }
        celija.setCellStyle(stil);
    }
}
\end{lstlisting}

\subsection{Formule}

Formule se unose metodom \texttt{setCellFormula()} koja prima string s formulom u Excel sintaksi, identičnoj onoj kakvu korisnik upisuje izravno u Excel. U primjeru se koristi formula \texttt{AVERAGE(D2:D6)} koja izračunava prosjek numeričkih ocjena unesenih u stupac D \cite{apachePOISpreadsheetQuickGuide}.

\begin{lstlisting}[style=java, caption={Unos formule i formatiranje retka}]
Row redFormule = list.createRow(zadnjiRed);

Cell celijaOznaka = redFormule.createCell(2);
celijaOznaka.setCellValue("Prosjek ocjena:");
celijaOznaka.setCellStyle(stilZbroj);

Cell celijaFormula = redFormule.createCell(3);
celijaFormula.setCellFormula("AVERAGE(D2:D6)");
celijaFormula.setCellStyle(stilZbroj);
\end{lstlisting}

\subsection{Automatska širina stupaca i zamrzavanje zaglavlja}

Apache POI nudi dvije korisne metode za poboljšanje preglednosti tablice. Metoda \texttt{autoSizeColumn()} automatski prilagođava širinu stupca najširem sadržaju u njemu, a zamrzava određenog broja redova ili stupaca metodom \texttt{createFreezePane()}, čime zaglavlje ostaje vidljivo pri pomicanju kroz dugačke tablice \cite{apachePOISpreadsheetQuickGuide}.

\begin{lstlisting}[style=java, caption={Automatska širina i zamrzavanje zaglavlja}]
for (int i = 0; i < stupci.length; i++) {
    list.autoSizeColumn(i);
}
list.createFreezePane(0, 1);
\end{lstlisting}

\subsection{Ugradnja grafa}

Uz tablične podatke, Apache POI omogućuje programsko kreiranje grafova koji su izravno vezani uz ćelije lista. Graf se kreira kroz \texttt{XSSFDrawing} objekt koji predstavlja sloj za crtanje na listu, a njegova pozicija i veličina definiraju se kroz \texttt{XSSFClientAnchor} parametar koji prima koordinate u obliku indeksa stupaca i redova \cite{apachePOISpreadsheetQuickGuide}.

\begin{lstlisting}[style=java, caption={Kreiranje stupčastog grafa}]
XSSFDrawing drawing = list.createDrawingPatriarch();
XSSFClientAnchor anchor = drawing.createAnchor(0, 0, 0, 0, 0, 8, 6, 25);

XSSFChart chart = drawing.createChart(anchor);
chart.setTitleText("Ocjene studenata");
chart.setTitleOverlay(false);

XDDFChartLegend legenda = chart.getOrAddLegend();
legenda.setPosition(LegendPosition.BOTTOM);

XDDFCategoryAxis xOs = chart.createCategoryAxis(AxisPosition.BOTTOM);
xOs.setTitle("Student");
XDDFValueAxis yOs = chart.createValueAxis(AxisPosition.LEFT);
yOs.setTitle("Ocjena");
yOs.setCrosses(AxisCrosses.AUTO_ZERO);

XDDFDataSource<String> imena = XDDFDataSourcesFactory.fromStringCellRange(
        list, new CellRangeAddress(1, 5, 0, 0));
XDDFNumericalDataSource<Double> ocjene = XDDFDataSourcesFactory
        .fromNumericCellRange(list, new CellRangeAddress(1, 5, 3, 3));

XDDFChartData podaciGrafa = chart.createData(ChartTypes.BAR, xOs, yOs);
podaciGrafa.setVaryColors(false);
XDDFChartData.Series serija = podaciGrafa.addSeries(imena, ocjene);
serija.setTitle("Ocjena", null);
((XDDFBarChartData) podaciGrafa).setBarDirection(BarDirection.COL);

chart.plot(podaciGrafa);
\end{lstlisting}

S \texttt{XDDFDataSourcesFactory.fromStringCellRange()} i \texttt{fromNumericCellRange()} metodama se definiraju izvori podataka za graf. Iste primaju referencu na list i raspon ćelija kroz \texttt{CellRangeAddress} objekt. U primjeru raspon \texttt{A2:A6} sadrži imena studenata kao kategorijsku os, a raspon \texttt{D2:D6} sadrži numeričke ocjene kao vrijednosnu os. Ključna prednost ovog pristupa je što graf nije statična slika nego živi objekt vezan uz podatke u ćelijama, pri svakom otvaranju datoteke u Excelu graf se automatski ažurira ako su se podaci promijenili.

\chapter{Zaključak}

Apache POI pokazao se kao pouzdana biblioteka za programski rad s Microsoft Office formatima u Java okruženju. Kroz praktične primjere u ovom radu vidljivo je kako generiranje strukturiranih Word i Excel dokumenata je izvedivo s relativno malo koda, uz punu kontrolu nad stilovima, sadržajem i formatiranjem. Najveća prednost biblioteke je platformska neovisnost, dokumenti se generiraju bez potrebe za instaliranim Microsoft Officeom, što je ključno za serverske i automatizacijske scenarije. Podrška za oba standarda, OLE2 i OOXML, osigurava kompatibilnost s dokumentima različitih generacija.

Ograničenja su ipak prisutna. SXSSF streaming način rada ograničen je isključivo na pisanje, a čitanje velikih datoteka zahtijeva SAX-temeljeni pristup koji je znatno složeniji za implementaciju. HSLF i XSLF moduli za PowerPoint nemaju zajedničko sučelje, što otežava pisanje koda neovisnog o formatu. Uz to, Word dokument interno može razbiti jedan vidljivi tekst u više XWPFRun objekata, što je potrebno uzeti u obzir pri implementaciji zamjene teksta, kao što je prikazano u primjeru cirkularnog pisma.

Najveći izazov pri radu s Apache POI bibliotekom bio je upravo problem razbijanja teksta na više XWPFRun objekata, što se pokazalo tek pri testiranju cirkularnog pisma kada zamjena placeholdera jednostavno nije radila kako je očekivano. Tek nakon što sam metodom pokušaja i pogreške shvatio da Word interno ne čuva tekst onako kako ga vidimo, nego ga dijeli na segmente koji ovise o formatiranju i internim optimizacijama, rješenje je postalo jasno. Taj detalj nije nigdje eksplicitno naglašen u službenoj dokumentaciji pa je traženje uzroka oduzelo više vremena nego sama implementacija. S druge strane, generiranje Excel izvještaja s formulama i stilovima teklo je glatko, što sugerira kako je Excel podrška u POI-u zrelija i bolje dokumentirana od Word podrške.

Važno je napomenuti kako postoji ograničenje vezano uz kompatibilnost XDDF grafova s LibreOffice Calc aplikacijom. Iako je generirani XML graf ispravan i u potpunosti u skladu s OOXML standardom, što je potvrđeno pregledom internog XML sadržaja putem \texttt{getCTChart().xmlText()} metode, LibreOffice ne renderira stupce grafa ispravno te prikazuje samo prazne osi. Isti graf ispravno se prikazuje u Microsoft Excelu i Google Sheetsu. Ovaj problem je nekompatibilnost između POI-jevog XDDF API-ja i LibreOfficeovog parsera OOXML grafova te nije uzrokovan greškom u kodu. Za produkcijska okruženja koja zahtijevaju prikaz u LibreOfficeu, preporučuje se zaobilaženje XDDF API-ja i direktno upisivanje XML strukture grafa kroz \texttt{getCTChartSpace().set()} metodu.

Apache POI preporučuje se za sve scenarije koji zahtijevaju automatiziranu obradu Office dokumenata u Java okruženju, od generiranja izvještaja i faktura do masovne personalizacije dokumenata u obrazovnim i poslovnim sustavima.

        String predlozakDatoteka = "src/main/java/com/zpracic/word/predlozak.docx";
        String datum = LocalDate.now()
                .format(DateTimeFormatter.ofPattern("dd.MM.yyyy."));

        try (BufferedReader br = new BufferedReader(
                new FileReader(csvDatoteka))) {
            String linija;
            boolean prviRed = true;
            while ((linija = br.readLine()) != null) {
                if (prviRed) {
                    prviRed = false;
                    continue;
                }
                String[] podaci = linija.split(",");
                String ime     = podaci[0];
                String prezime = podaci[1];
                String jmbag   = podaci[2];
                String kolegij = podaci[3];
                String ocjena  = podaci[4];

                try (FileInputStream fis = new FileInputStream(predlozakDatoteka);
                     XWPFDocument dokument = new XWPFDocument(fis)) {

                    for (XWPFParagraph paragraf : dokument.getParagraphs()) {
                        zamijeniUParagrafu(paragraf, ime, prezime, 
                                jmbag, kolegij, ocjena, datum);
                    }

                    String izlazDatoteka = "src/main/java/com/zpracic/word/obavijest_"
                            + ime + "_" + prezime + ".docx";
                    try (FileOutputStream out =
                            new FileOutputStream(izlazDatoteka)) {
                        dokument.write(out);
                    }
                    System.out.println("Kreiran: " + izlazDatoteka);
                }
            }
        }
        System.out.println("Cirkularno pismo uspjesno generirano.");
    }

    private static void zamijeniUParagrafu(XWPFParagraph paragraf,
            String ime, String prezime, String jmbag,
            String kolegij, String ocjena, String datum) {

        StringBuilder cijeli = new StringBuilder();
        for (XWPFRun run : paragraf.getRuns()) {
            if (run.getText(0) != null) {
                cijeli.append(run.getText(0));
            }
        }
        String tekst = cijeli.toString();
        tekst = tekst.replace("{{IME}}", ime);
        tekst = tekst.replace("{{PREZIME}}", prezime);
        tekst = tekst.replace("{{JMBAG}}", jmbag);
        tekst = tekst.replace("{{KOLEGIJ}}", kolegij);
        tekst = tekst.replace("{{OCJENA}}", ocjena);
        tekst = tekst.replace("{{DATUM}}", datum);

        if (!paragraf.getRuns().isEmpty()) {
            paragraf.getRuns().get(0).setText(tekst, 0);
            for (int i = 1; i < paragraf.getRuns().size(); i++) {
                paragraf.getRuns().get(i).setText("", 0);
            }
        }
    }
}
